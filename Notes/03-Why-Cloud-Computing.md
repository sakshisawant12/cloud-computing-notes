# ☁️ Chapter 3 - Why Cloud Computing?

## Why was Cloud Computing introduced?

Before Cloud Computing, companies faced many challenges:

- 💰 High infrastructure cost
- ⏳ Took weeks or months to set up servers
- 👨‍💻 Required IT staff to maintain infrastructure
- 📈 Difficult to increase or decrease resources
- 🔧 Hardware failures could affect applications

Cloud Computing solves these problems by providing resources over the Internet.

---

## Example

Imagine you're starting an online shopping website.

### ❌ Without Cloud (On-Premises)

You need to:

- Buy servers
- Buy storage
- Buy networking devices
- Build a server room
- Hire IT engineers

💰 This may cost lakhs or even crores of rupees before your website is live.

---

### ✅ With Cloud

You simply:

- Sign in to a cloud platform
- Launch a virtual server
- Upload your website
- Start using it

🚀 It takes only a few minutes.

---

# Why Companies Prefer Cloud

## 1. Lower Cost 💰

No need to buy expensive hardware.

You only pay for the resources you use.

### Example

Instead of buying a ₹5 lakh server, you can rent computing resources and pay only for your usage.

---

## 2. Pay As You Go 💳

You pay only for the resources you use.

### Example

Like an electricity bill—you pay based on your consumption, not a fixed amount.

---

## 3. Faster Deployment ⚡

Resources can be created within minutes.

Earlier, setting up infrastructure could take days or even weeks.

---

## 4. Scalability 📈

Scalability means increasing or decreasing resources based on business needs.

### Example

If your website receives more visitors, you can increase CPU, RAM, or add more servers.

---

## 5. Elasticity 🤖

Elasticity means resources automatically increase or decrease based on demand.

### Example

During a festival sale:

- More users → More resources are added automatically.
- Sale ends → Extra resources are removed automatically.

### Difference

| Scalability | Elasticity |
|-------------|------------|
| Manual or planned increase/decrease | Automatic increase/decrease |
| Based on business requirements | Based on real-time demand |

---

## 6. High Availability 🌍

High Availability means your application remains available even if one server fails.

### Example

If one server goes down, another server continues serving users.

---

## 7. Global Access 🌎

Users can access applications from anywhere using the Internet.

### Examples

- Gmail
- Google Drive
- Netflix

---

## 8. Backup & Disaster Recovery 💾

Cloud providers offer backup and disaster recovery solutions.

If one server or data center fails, data can be restored from another location.

---

## Real-Life Example

### 🍽️ Opening a Restaurant

### ❌ Without Cloud

You need to:

- Buy land
- Build the restaurant
- Buy furniture
- Hire staff

It takes months before you can serve customers.

### ✅ With Cloud

It's like renting a fully furnished restaurant.

You can start serving customers almost immediately.

---

# Summary

| Problem | Cloud Solution |
|----------|----------------|
| High hardware cost | Pay only for what you use |
| Slow setup | Deploy in minutes |
| Difficult to scale | Easy scalability |
| Hardware maintenance | Managed by the cloud provider |
| Downtime | High Availability |
| Data loss | Backup & Disaster Recovery |

---

- Cloud reduces infrastructure costs.
- Resources are available within minutes.
- Pay only for what you use.
- Easy to scale resources.
- Automatic resource management through elasticity.
